Ledebe Protector Docs

Ledebe Protector detects and masks personally identifiable information (PII) before you share text with AI tools, colleagues, or external services. Your original data never leaves your device.

🛡️ Overview

Ledebe Protector works by replacing sensitive information in your text with safe placeholders, letting you share or process the text freely. You can then restore the original content at any time.

Supported PII types:

  • Emails — e.g. john@example.com → [LDB_EMAIL#1]
  • Phone numbers — local and international formats
  • IP addresses — IPv4 and IPv6
  • Credit/debit card numbers
  • IBAN / bank account numbers
  • NIN / BVN — national identity numbers
  • Custom terms — any word or phrase you define

🚀 How to Use Ledebe

New to Ledebe? Follow these steps to protect and share text safely in under a minute.

Step 1 — Type or paste your text

Open the app and type (or paste) any text that contains sensitive information into the main input area. It could be an email, a support ticket, a contract excerpt — anything.

ℹ️
Example: "Hi, my name is Sarah Jones, email sarah@company.com and my phone is 07911 123456."

Step 2 — Click Send / Protect

Click the Send button (or press Ctrl + Enter / ⌘ Cmd + Enter on Mac). Ledebe automatically scans your text and replaces every piece of detected PII with a safe placeholder.

💡
The result will look like: "Hi, my name is [LDB_CUSTOM1], email [LDB_EMAIL1] and my phone is [LDB_PHONE1]." — safe to share anywhere.

Step 3 — Switch between views

Each message has two tabs beneath it:

  • Plain Text — your original, unmasked text (visible only to you).
  • Protected Text — the masked version with placeholders, safe to copy and share.

Step 4 — Copy the protected text

Click the 📋 Copy button on the message, or press Ctrl/⌘ Cmd + Shift + X, to copy the protected version to your clipboard. Paste it into ChatGPT, an email, a support system — wherever you need it.

Step 5 — Restore the original (optional)

If you receive a reply that contains placeholders (e.g. from an AI tool), paste it back into Ledebe. It will automatically swap the placeholders back to the real values, so you can read the response with your actual data.

Step 6 — Add custom terms (optional)

Have specific words you always want to hide — like your company name, a project code, or a client name? Add them to Custom Terms in the sidebar. They'll be masked automatically every time.

💡
Quick tip: As you type, Ledebe suggests words to add as custom terms. If a word is already protected, it shows a green "already protected" chip so you never add the same term twice.

Switching between Protect and Ask AI

At the bottom of the screen, just above the text box, you'll see a mode toggle:

🛡️ Protect 🤖 Ask AI
  • 🛡️ Protect (default) — masks your text and shows you the safe version to copy. Nothing is sent to any AI. Use this when you just want to sanitise text before pasting it elsewhere.
  • 🤖 Ask AI — sends your message directly to an AI (OpenAI, Anthropic, or Google) with PII already masked. The AI replies, and you see both the AI's response and a re-masked "Safe to Share" version. Use this for a full private AI chat experience.
💡
You can switch modes at any time — even mid-conversation. The mode toggle remembers your last choice.

💻 System Requirements

  • macOS: macOS 11 (Big Sur) or later for Apple Silicon · macOS 10.15 (Catalina) or later for Intel
  • Windows: Windows 10 or later (64-bit)
  • Linux: Ubuntu 18.04+ or any distro that supports AppImage
  • RAM: 4 GB minimum, 8 GB recommended
  • Disk: ~300 MB free space
  • Internet: Only required for AI chat features (API calls)

🍎 macOS Apple Silicon

For M1, M2, and M3 Macs. Download the arm64.dmg file.

Installation steps

  1. Download Ledebe Protector-1.0.0-arm64.dmg from the Downloads page.
  2. Open the downloaded .dmg file from your Downloads folder.
  3. Drag the Ledebe Protector app into your Applications folder.
  4. Eject the disk image and open the app from Applications or Spotlight.
⚠️
Gatekeeper warning: On first launch, macOS may show "Ledebe Protector cannot be opened because it is from an unidentified developer." To bypass: right-click the app icon → click Open → click Open in the dialog. You only need to do this once.
💡
Alternatively, go to System Settings → Privacy & Security and click Open Anyway next to the Ledebe Protector entry.

🍎 macOS Intel

For Intel-based Macs. Download the x64.dmg file.

Installation steps

  1. Download Ledebe Protector-1.0.0-x64.dmg from the Downloads page.
  2. Open the .dmg file and drag the app to Applications.
  3. Launch from Applications or Spotlight.
⚠️
Same Gatekeeper behaviour as Apple Silicon — right-click → Open on first launch if macOS blocks it.

🪟 Windows 10 / 11

Download the Ledebe Protector Setup 1.0.0.exe NSIS installer. This creates Start Menu shortcuts, a desktop icon, and an uninstaller.

Installation steps

  1. Download Ledebe Protector Setup 1.0.0.exe from the Downloads page.
  2. Run the installer — you may see a Windows SmartScreen warning ("Windows protected your PC"). Click More info then Run anyway.
  3. Follow the setup wizard (Next → Install → Finish).
  4. Launch Ledebe Protector from the Start Menu or desktop shortcut.
⚠️
SmartScreen warning is normal for newly published apps without an EV code signing certificate. The app is safe — click More info → Run anyway to proceed.
ℹ️
To uninstall: go to Settings → Apps, find Ledebe Protector, and click Uninstall.

🐧 Linux AppImage

Ledebe Protector ships as a universal AppImage — no installation required. It runs on Ubuntu, Debian, Fedora, Arch, and most other distros.

Installation steps

  1. Download Ledebe Protector-1.0.0.AppImage from the Downloads page.
  2. Open a terminal and navigate to your Downloads folder:
    cd ~/Downloads
  3. Make the file executable:
    chmod +x "Ledebe Protector-1.0.0.AppImage"
  4. Run the app:
    ./"Ledebe Protector-1.0.0.AppImage"
💡
You can also right-click the AppImage in your file manager → Properties → Permissions → check "Allow executing file as program", then double-click to run.
ℹ️
Some distros require libfuse2 for AppImages. If the app fails to launch, install it:
Ubuntu/Debian: sudo apt install libfuse2
Fedora: sudo dnf install fuse-libs

🔒 Protecting Text

  1. Paste or type your text into the input area at the bottom of the screen.
  2. Click Send. Ledebe scans and replaces all detected PII with placeholders like [LDB_EMAIL1], [LDB_PHONE1].
  3. Each message shows two tabs — AI's Response and Safe to Share. Switch between them to see the original or protected version.
  4. Click 📋 Copy on the message to copy the protected text — safe to paste into any AI tool.
  5. To restore original values, paste the protected text back and click Restore.
💡
Placeholders are consistent within a session — the same email always maps to the same placeholder, so your text structure is preserved.

✋ Highlight to Protect / Unprotect

You can protect or unprotect individual words directly from any sent message — without retyping anything.

Protect a word

  1. Highlight any word or phrase inside a sent message bubble.
  2. A floating button appears above your selection: 🛡️ + Protect
  3. Click it — the word is added to your Custom Terms and the message is re-protected immediately.

Unprotect a word

  1. Highlight a word that is already in your protected terms.
  2. The floating button now shows: 🔓 Unprotect (in red)
  3. Click it — the word is removed from Custom Terms and the message updates instantly to show it unmasked.
ℹ️
This works inside chat messages and inside the document preview. Changes take effect immediately — no need to re-send the message.

⚙️ Custom Terms

Add your own words or phrases to always protect — names, company names, project codes, anything sensitive.

Adding a term

  1. Open the Custom Terms panel in the sidebar (bottom left).
  2. Type your term and click Save & Close.
  3. Every future message will automatically mask it as [LDB_CUSTOM1].
  4. Terms are saved locally and persist across sessions.

Searching your terms

If you have many terms saved, use the built-in search to find them quickly.

  1. Click the 🔍 icon in the Custom Terms header.
  2. A search popup opens showing all saved terms.
  3. Type to filter — matching characters are highlighted in amber.
  4. Click ✕ next to any term to delete it instantly.
💡
The search popup shows a count e.g. "4 of 47 terms" so you always know how many terms match your query.

📂 File Upload

Ledebe can process entire documents — not just typed text. Upload a file and it extracts, protects, and previews the content for you.

How to upload

  1. Click the + button in the input area (bottom left of the text box).
  2. Select a file from your device.
  3. Ledebe processes it and shows a document card with the file name, word count, and number of items protected.

Supported file types

  • .txt / .csv — text extracted and protected instantly
  • .docx — full Word document processed and previewed
  • .pdf — text extracted via PDF.js, then protected
  • Images (JPG, PNG, etc.) — OCR extracts text from the image, then it's protected

Document preview

After uploading, click the document card to open the preview modal. It shows two side-by-side views:

  • Original — your document as uploaded
  • Protected — all PII replaced with placeholders

From the preview you can 📥 Download the protected version as a file.

ℹ️
You can also highlight text inside the document preview to trigger the 🛡️ Protect / 🔓 Unprotect floating button, just like in chat messages.

🤖 AI Chat

Switch to Ask AI mode to send questions directly to an AI. Ledebe masks your PII before the message is sent, so the AI never sees your real data.

Setup

  1. Click ⚙️ Settings in the top bar.
  2. Enter your API key for the provider you want to use:
    • OpenAI — ChatGPT (GPT-4, GPT-3.5)
    • Anthropic — Claude
    • Google — Gemini
  3. Toggle the mode pill at the bottom of the screen to 🤖 Ask AI.
  4. Type your message and send — PII is masked automatically before it leaves your device.
ℹ️
API keys are stored locally only — they are never sent to Ledebe servers.

📋 History & Search

Every conversation is automatically saved to your local history. The sidebar on the left lists all past chats.

Using history

  • Click any history item to reload that full conversation.
  • Hover over an item → click ⋯ → Rename or Delete.
  • Click + New Chat at the top of the sidebar to start fresh.

Searching history with ⌘K

Press ⌘K (Mac) or Ctrl+K (Windows) to open the command palette — a fast search over all your past chats.

  1. Press ⌘K — the search palette slides in.
  2. Start typing — results filter instantly, with matching text highlighted.
  3. Use ↑↓ arrow keys to navigate results.
  4. Press Enter to open the selected chat.
  5. Press Esc to close without selecting.
💡
You can also click the Search history button at the top of the sidebar to open the same palette.

🧩 VS Code Extension

Ledebe is also available as a VS Code extension — built for developers who want to protect sensitive data before sharing code, logs, or config files with AI tools like GitHub Copilot or ChatGPT.

Install

  1. Open VS Code.
  2. Go to the Extensions panel (left sidebar).
  3. Search "Ledebe Protector".
  4. Click Install.

Features

  • Right-click → 🛡️ Protect with Ledebe — masks PII in any selected text inline
  • Right-click → 🔓 Restore Original — swaps placeholders back to real values
  • Right-click → ➕ Add to Protected Terms — adds the selected word to your custom terms list
  • Sidebar panel — full protect/restore workflow with custom terms management
  • Auto-detect on file open — warns you when PII is found in .env, .log, .json, .csv files

Keyboard shortcut

Action Shortcut
Protect selected text ⌘⇧L Mac  /  Ctrl+Shift+L Windows
💡
Custom terms added in the VS Code extension are saved per VS Code profile — separate from the desktop app's terms list.

🔧 Troubleshooting

App crashes on launch

Crash reports are stored at ~/.ledebe-crashes/. If you need support, include the latest crash log when contacting us.

PII not being detected

Ensure the text format matches a supported pattern. For unusual formats (e.g. non-standard phone formats), add the value as a Custom Term.

AI chat not responding

  • Check that your API key is correct in Settings.
  • Verify you have an active internet connection.
  • Check your API quota/limits with the provider.

Dark mode not saving

Dark mode preference is stored in local storage. Clearing your browser cache (in Electron: Settings → Clear Data) will reset it.
